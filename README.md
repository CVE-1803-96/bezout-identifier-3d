# Universal Math Visualization Engine

An interactive **3D math visualization** built with **React + Vite + TypeScript + Tailwind** and rendered via **Three.js** (`@react-three/fiber` / `@react-three/drei`).

Right now, the project ships a **Bézout’s Identity Visualizer**: it shows the surface $z=ax+by$, the plane $z=\gcd(a,b)$, and highlights integer pairs $(x,y)$ that satisfy:

$$ax + by = \gcd(a,b)$$

## Features

- **Real-time parameters**: adjust integers `a`, `b`, and view range.
- **3D scene**:
  - Surface: $z=ax+by$
  - Plane: $z=\gcd(a,b)$
  - Intersection line: $ax+by=\gcd(a,b)$
  - Marked integer solutions $(x,y)$ in the chosen range
- **Camera controls**: drag to rotate, scroll to zoom, right-click to pan.

## How it works (math)

- **`gcd(a,b)`** is computed with the Euclidean algorithm.
- A **particular solution** $(x_0,y_0)$ to $ax+by=\gcd(a,b)$ is found using the **Extended Euclidean Algorithm**.
- All integer solutions are generated by:

$$x = x_0 + \frac{b}{g}t,\qquad y = y_0 - \frac{a}{g}t,\qquad t\in\mathbb{Z}$$

and then filtered to those within the chosen view range.

Implementation lives in `src/utils/math.ts`.

## Project structure

- **`index.html`**: app shell
- **`src/main.tsx`**: React entrypoint
- **`src/App.tsx`**: UI + parameter controls + results panel
- **`src/components/BezoutScene.tsx`**: Three/Fiber scene rendering
- **`src/utils/math.ts`**: `gcd`, `extendedGcd`, and solution generation
- **`vite.config.ts`**: Vite config (+ `@` alias to `src/`, single-file plugin)

## Tech stack

- React 19, TypeScript, Vite
- Tailwind CSS (via `@tailwindcss/vite`)
- Three.js + React Three Fiber + Drei

## Notes

- Inputs allow negative `a` / `b`. The implementation normalizes the returned gcd to be **non-negative**.
- If you want to add more “math scenes,” the pattern is to create new components under `src/components/` and wire them up in `src/App.tsx`.
